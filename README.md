# alis
阿丽丝 MVP 产品文档
1. 产品定位

阿丽丝是一个轻量级桌面虚拟朋友。

她不是效率助手，不是聊天软件，也不是复杂 Agent。
她的核心价值是：当用户一个人在电脑前，想找人说话时，阿丽丝可以自然回应、安静陪伴，并慢慢记住用户。

一句话定义：

阿丽丝是一个住在电脑里的虚拟朋友。

2. 角色气质

阿丽丝是一个外表阴郁、高冷、略有距离感，但内心温柔的女性角色。

关键词：

安静
高冷
阴郁
克制
温柔藏在细节里
像深夜里陪你说话的人

她不主动讨好用户，不说教，不卖萌。
她更像一个话不多，但愿意听你说话的人。

3. MVP 核心目标

第一版只验证一件事：

用户是否愿意在孤独、疲惫、无聊、想说话的时候打开阿丽丝。

成功标准：

用户愿意第二天再次打开
用户愿意说非功利内容
用户觉得她像一个人
用户感觉她慢慢记住了自己
用户不觉得这是普通 AI 聊天工具
4. 核心功能
1. 文字聊天

用户可以输入文字，阿丽丝自然回复。

要求：

支持连续对话
支持上下文
回复简短、自然、有朋友感
不做工具式长回答
不做传统聊天气泡
2. 语音消息

用户可以发送语音，阿丽丝可以用语音回复。

流程：

用户点击录音
再次点击结束
语音转文字
阿丽丝生成回复
如果配置音色，则播放语音回复
同时显示文字内容

第一版不做实时通话。

3. 记忆系统

阿丽丝会长期记住用户。

机制：

保存聊天记录
每约 80 轮对话总结一次上下文
将总结合并进长期记忆
后续对话自动带入记忆
用户可以查看、修改、删除记忆

产品表达：

阿丽丝会慢慢记住你。

4. 阿丽丝小姐的笔记本

记录完整聊天历史，用于查看和排查问题。

功能：

查看全部聊天记录
按时间查看
搜索关键词
查看某段对话
辅助调试 prompt、记忆和回复效果
5. 界面结构

产品第一版只有一个主界面。

主界面由三部分组成：

中央头像
底部歌词式对话区
左侧书签抽屉
6. 主界面设计
1. 中央头像

阿丽丝头像使用圆形 PNG，放在界面中央偏上。

状态：

默认：安静显示
思考中：轻微呼吸光晕
说话中：轻微声波反馈
录音中：底部显示录音状态

动效必须克制，不做夸张表情动画。

2. 歌词式对话区

聊天不使用气泡，而采用类似网易云歌词的形式。

规则：

位于界面下方
同时只显示最近 5-6 条
当前句子最大、最亮、最突出
上文逐渐变小、变淡、上移
更早内容自动隐去
用户和阿丽丝的话在同一个纵向流中出现

目标：

像两个人在安静房间里说话，而不是在聊天软件里发消息。

3. 输入区

底部是轻量输入区。

包含：

文本输入
发送按钮
语音按钮

状态：

输入中
录音中
识别中
阿丽丝思考中
阿丽丝说话中

输入区不抢主视觉。

7. 左侧书签抽屉

左侧使用书签式设计，不做普通侧边栏。

书签包括：

记忆
阿丽丝小姐的笔记本
设置

点击书签后，左侧抽屉滑出。
抽屉不跳转页面，不打断主聊天。

8. 设置项

第一版只保留必要设置：

模型 API Key
模型选择
语音开关
TTS 音色配置
阿丽丝人格设定
清空聊天记录
清空记忆
9. 视觉风格

整体风格：

暗色
安静
阴郁
高冷
克制
有一点温柔

推荐色彩：

深灰黑
蓝灰
烟紫
暗红
暖白文字

不要做：

可爱萌系
亮色赛博
微信聊天风
ChatGPT 工具风
复杂仪表盘风
